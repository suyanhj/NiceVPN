# Proposal: vpn-bugfix-2026-04-03

## 背景

上一轮 bugfix（vpn-bugfix-2026-04）完成后，实际使用中发现多处 UI 和业务逻辑问题，涵盖用户管理、组管理、防火墙规则、服务管理四个模块，以及一个启动时的运行时类型错误。

## 问题清单

### 运行时错误
- `CertService._get_index_path` 中 `Path(self._config.pki_dir)` 在 `pki_dir` 未配置时传入 `Box({})` 对象，导致 `TypeError: expected str, bytes or os.PathLike object, not Box`

### 用户管理
1. **状态栏排版**：用户卡片信息分成上下两行（主区 + 侧栏），不够紧凑，要求全部水平排在一行
2. **搜索 bug**：初始列表未渲染在 `list_container` 内，搜索时新内容在容器内、旧内容在容器外，导致"双重显示"
3. **虚拟 IP 分配**：用 `user_count` 做偏移量分配 IP，删除用户后计数不准，可能分配重复 IP

### 组管理
4. **默认组逻辑**：硬编码"默认用户组"保护不灵活。需求是"第一个组为根组，后续组必须是根组的子网"

### 防火墙
5. **新建规则表单**：源 IP / CIDR 归属类型搞反了；用户类型应下拉多选而非手填 ID；OpenVPN 实例字段多余（单机）；目标 IP 应支持多个
6. **搜索**：搜索结果和原列表分开显示
7. **拖拽排序**：SortableJS 未加载，拖拽不生效

### 服务管理
8. **日志功能**：前端"查看日志"按钮只有占位提示，需读取 `log-append` 配置实现真正的日志查看

## 目标

修复以上全部问题，不引入新功能，不改变数据结构根本设计。

## 非目标

- 不引入外部数据库
- 不新增页面
- 不改动初始化向导流程
