# 前端 CSS 统一重构

## 背景

各管理页面（用户、组、证书、服务、设置、防火墙）的 CSS 样式存在大量重复定义，
各页面曾用各自前缀（`user-`、`group-`、`cert-`、`service-`、`settings-`；**防火墙**现为 `firewall-*` 专名 + 与**组管理等同构**的 `mgmt-*`）
定义了相同结构的组件样式（页面容器、面板、标题栏、工具栏、列表、卡片、按钮等），
导致 `theme.py` 膨胀到约 2981 行，维护成本高、改动容易遗漏。

## 目标

1. 提取通用管理页面组件为 `mgmt-*` 前缀的共享 CSS 类
2. 各页面 Python 文件统一使用 `mgmt-*` 类名
3. 保留各页面确实独有的样式（如证书高级面板、服务状态指示器等）
4. 删除不再使用的 `drag_list.py` 组件
5. 纯样式重构，不涉及任何业务逻辑变动

## 约束

- 不修改任何业务逻辑
- 不改变页面功能和交互行为
- 保持各页面视觉效果一致
